# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

Commits follow [Conventional Commits](https://www.conventionalcommits.org/),
enforced by the `commit-msg` Husky hook.

## How releases are cut

This template does not ship an automated release pipeline. When a project
needs one, adopt [release-please](https://github.com/googleapis/release-please)
or [changesets](https://github.com/changesets/changesets) — both work well with
the Conventional Commits used here. Until then, update this file by hand on
each release and tag the commit.
